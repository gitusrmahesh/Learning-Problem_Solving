Conditional Statements

Conditional statements allow you to execute different blocks of code based on whether a certain condition is true or false. They are the fundamental decision-making structures in programming. This section provides an overview of C#'s primary conditional constructs and an interactive demo to see them in action.

The if-else Statement

The most common conditional statement. It executes a block of code if a condition is true, and an optional else block if it's false.

int age = 20;
if (age >= 18)
{
    Console.WriteLine("You are an adult.");
}
else
{
    Console.WriteLine("You are a minor.");
}

The switch Statement

Useful when you have a single variable to compare against multiple possible constant values.

char grade = 'B';
switch (grade)
{
    case 'A':
        Console.WriteLine("Excellent!");
        break;
    case 'B':
        Console.WriteLine("Good");
        break;
    default:
        Console.WriteLine("Needs improvement");
        break;
}

Looping Constructs

Loops are used to execute a block of code repeatedly as long as a certain condition is met. They are essential for iterating over collections of data, such as arrays or lists, and for performing repetitive tasks without writing the same code over and over. This section explores the different types of loops available in C#.

for Loop

Best when you know the exact number of iterations.

for (int i = 0; i < 5; i++)
{
  // Code runs 5 times
}

while Loop

Runs as long as a condition is true. The condition is checked *before* each iteration.

int n = 0;
while (n < 5)
{
  n++;
}

do-while Loop

Similar to `while`, but the condition is checked *after* each iteration. Guarantees at least one execution.

int n = 0;
do
{
  n++;
} while (n < 5);

foreach Loop

The simplest way to iterate over all elements in a collection.

var numbers = new[] {1,2,3};
foreach (var num in numbers)
{
  //...
}

Complexity Analysis (Big O Notation)

Algorithmic complexity is a measure of how much time or memory an algorithm requires to run as its input size grows. We use "Big O" notation to describe this relationship. Understanding Big O is crucial for writing efficient and scalable code. This section introduces common complexity classes and visualizes their growth rates.

O(1) - Constant Time

The algorithm takes the same amount of time regardless of the input size. Example: Accessing an array element by its index.

O(n) - Linear Time

The runtime grows linearly with the input size 'n'. Example: Iterating through an entire list once.

O(n²) - Quadratic Time

The runtime is proportional to the square of the input size. Example: A nested loop iterating over the same list.
