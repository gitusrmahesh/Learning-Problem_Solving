Kadane's Algorithm

An Interactive Guide to the Maximum Subarray Problem

1. The Problem: Maximum Subarray Sum

Given an integer array, the goal is to find the contiguous subarray (containing at least one number) which has the largest sum and return its sum.

For example, consider the array below. The subarray [4, -1, 2, 1] has the largest sum of 6.

-2 1 -3
4 -1 2 1
-5 4

2. The Brute Force Approach

The simplest way to solve this is to check every possible subarray. We can use two nested loops to define the start and end of a subarray, and a third loop to calculate its sum. This approach has a time complexity of O(n³), which is very inefficient for large arrays.

A slight optimization is to calculate the sum in the second loop, reducing the complexity to O(n²). However, for large inputs, this is still too slow. This is where Kadane's Algorithm comes in, providing a brilliant O(n) solution.

3. Kadane's Algorithm: The Core Idea

Kadane's algorithm is a dynamic programming approach that solves this problem in a single pass. The key idea is to iterate through the array and, at each position, make a decision:

  1. We maintain a `current_max` sum, which is the maximum sum of a subarray ending at the current position.
  2. For each element, we either extend the previous subarray by adding the current element to `current_max`, OR we start a new subarray with just the current element. We choose whichever is larger.
  3. We also maintain a `global_max` sum, which is the maximum sum found so far across the entire array. After calculating `current_max` at each step, we update `global_max` if `current_max` is larger.

Essentially, if at any point the `current_max` becomes negative, it's better to discard the previous subarray and start a new one from the next element, because a negative prefix will only decrease the sum of any future subarray.

5. C# Implementation

Here is a concise implementation of Kadane's algorithm in C#.

public class Solution {
    public int MaxSubArray(int[] nums) {
        // If the array is empty, there's no subarray
        if (nums == null || nums.Length == 0) {
            return 0;
        }

        // Initialize both maxes with the first element
        int currentMax = nums[0];
        int globalMax = nums[0];

        // Start from the second element
        for (int i = 1; i < nums.Length; i++) {
            // Decide whether to extend the current subarray or start a new one
            currentMax = Math.Max(nums[i], currentMax + nums[i]);
            
            // Update the global max if the new current max is greater
            if (currentMax > globalMax) {
                globalMax = currentMax;
            }
        }

        return globalMax;
    }
}

6. Variation: Maximum Circular Subarray Sum

A common variation is finding the maximum subarray sum in a circular array, where the end of the array connects to the beginning.

This problem has two cases:

  1. The maximum subarray is a standard one (not wrapping around). This can be found with the normal Kadane's algorithm.
  2. The maximum subarray wraps around. This means the subarray consists of a prefix and a suffix of the array. The sum of this wrapping subarray is equal to the Total Sum of the Array minus the sum of the non-wrapping minimum subarray in the middle.

So, the strategy is:
1. Find the maximum subarray sum using Kadane's (Case 1).
2. Find the minimum subarray sum by inverting the signs of all array elements and running Kadane's again. Then calculate `Total Sum - Minimum Subarray Sum` (Case 2).
3. The answer is the maximum of these two cases. (A special edge case is when all numbers are negative; the answer is simply the largest number).

C# Code for Circular Variation

public int MaxSubarraySumCircular(int[] nums) {
    int totalSum = 0;
    int currentMax = 0;
    int globalMax = int.MinValue;
    int currentMin = 0;
    int globalMin = int.MaxValue;

    foreach (int num in nums) {
        totalSum += num;

        // Standard Kadane's for max sum
        currentMax = Math.Max(currentMax + num, num);
        globalMax = Math.Max(globalMax, currentMax);

        // Kadane's for min sum
        currentMin = Math.Min(currentMin + num, num);
        globalMin = Math.Min(globalMin, currentMin);
    }
    
    // If totalSum equals globalMin, it means all numbers are negative.
    // In this case, the non-circular max is the answer.
    if (globalMax > 0) {
        return Math.Max(globalMax, totalSum - globalMin);
    } else {
        return globalMax;
    }
}
