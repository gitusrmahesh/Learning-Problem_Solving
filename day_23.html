Day 23: Anagrams & Frequency Maps

What are Anagrams?

An anagram is a word or phrase formed by rearranging the letters of a different word or phrase, typically using all the original letters exactly once. For example, "listen" is an anagram of "silent".

While this concept is simple, checking for anagrams programmatically presents interesting challenges. The most efficient and versatile approach involves a data structure called a Frequency Map (or a character count array). This map stores each character from a string and its corresponding count, providing a unique "signature" for the string's character composition.

C# Code Walkthrough

This O(n) solution is highly efficient. It builds a frequency map for the first string and then decrements counts using the second string. If all counts return to zero, they are anagrams.

public bool IsAnagram(string s, string t) {
    if (s.Length != t.Length) return false;

    var counts = new Dictionary<char, int>();

    // Build frequency map for s
    foreach (char c in s) {
        counts[c] = counts.GetValueOrDefault(c, 0) + 1;
    }

    // Decrement counts using t
    foreach (char c in t) {
        if (!counts.ContainsKey(c) || counts[c] == 0) {
            return false;
        }
        counts[c]--;
    }

    return true;
}

This O(n log n) solution is very intuitive. If two strings are anagrams, their sorted versions will be identical.

public bool IsAnagramSorting(string s, string t) {
    if (s.Length != t.Length) return false;

    var sArray = s.ToCharArray();
    var tArray = t.ToCharArray();

    Array.Sort(sArray);
    Array.Sort(tArray);

    return new string(sArray) == new string(tArray);
}

Problem 2: Group Anagrams

Interactive Demo

C# Code Walkthrough

The key is to generate a canonical representation (a unique signature) for each word. Words that are anagrams will have the same signature. We use a dictionary to group words by this signature.

public IList<IList<string>> GroupAnagrams(string[] strs) {
    var groups = new Dictionary<string, IList<string>>();

    foreach (var str in strs) {
        // Create the signature: a sorted string
        char[] charArray = str.ToCharArray();
        Array.Sort(charArray);
        string key = new string(charArray);

        if (!groups.ContainsKey(key)) {
            groups[key] = new List<string>();
        }
        groups[key].Add(str);
    }

    return new List<IList<string>>(groups.Values);
}

Performance Analysis

When checking for anagrams, the frequency map approach is generally superior to sorting. Let 'N' be the length of the strings. The frequency map method takes O(N) time because it iterates through each string once. The sorting method takes O(N log N) time due to the sorting algorithm's complexity. For grouping 'K' strings of average length 'N', the complexity is O(K * N log N) using sorting and O(K * N) using a frequency map as a key.
