Debugging Techniques in C#

Debugging is the process of finding and fixing errors. While `Console.WriteLine()` is a start, Visual Studio's built-in debugger is far more powerful. It allows you to pause your program, inspect its state, and execute it line by line.

🔴 Breakpoints

A breakpoint pauses your program's execution right *before* a specific line of code is run. This allows you to "freeze" time and inspect the values of variables at that exact moment. To set one, simply click in the margin next to the line number in Visual Studio.

Stepping Through Code

Once paused at a breakpoint, you can control the flow of execution:

  • F10 (Step Over) Executes the current line. If it's a method call, it runs the entire method without showing you the internal steps.
  • F11 (Step Into) Executes the current line. If it's a method call, the debugger jumps *inside* that method, allowing you to debug it.
  • Shift+F11 (Step Out) Finishes executing the current method and returns to the line where it was called.

Inspecting Variables

While paused, you can easily check variable values. Hover your mouse over any variable to see its current value. For more detail, use the 'Locals', 'Autos', and 'Watch' windows at the bottom of Visual Studio, which provide a live view of your application's state.

Handling Corner Cases

Edge cases are inputs at the extremes of the valid range (e.g., zero, null, empty, min/max values). Failing to consider them is a top source of bugs. Robust code anticipates and handles these scenarios gracefully.

Empty/Null Arrays

A function that processes a collection should never assume it contains items. It might be empty or even `null`.

Problem: Unsafe Code

public double CalculateAverage(int[] numbers) { // Throws NullReferenceException if numbers is null. // Throws DivideByZeroException if numbers is empty. return numbers.Sum() / (double)numbers.Length; }

Solution: Robust Code

public double CalculateAverage(int[] numbers) { if (numbers == null || numbers.Length == 0) { return 0; // Return a safe default value. } return numbers.Sum() / (double)numbers.Length; }

Large Numbers & Overflow

Calculations involving large numbers can exceed the maximum value for a data type (like `int`), causing an overflow where the value wraps around unexpectedly.

Problem: Integer Overflow

public int AddLargeNumbers(int a, int b) { // If a = 2_000_000_000 and b = 2_000_000_000, // the result will be incorrect due to overflow. return a + b; }

Solution: Use Larger Type

public long AddLargeNumbers(int a, int b) { // Use 'long' for the calculation and return type // to accommodate a much larger result. return (long)a + b; }

Negative Numbers

Functions expecting positive inputs (like for array indices or factorials) can fail if they receive a negative number.

Problem: Invalid Input

public int GetNthElement(int[] array, int n) { // If n is -1, this will throw an // IndexOutOfRangeException. return array[n]; }

Solution: Input Validation

public int GetNthElement(int[] array, int n) { // Validate the input before using it. if (n < 0 || n >= array.Length) { throw new ArgumentOutOfRangeException(nameof(n)); } return array[n]; }

Edge Case Practice Problems

Test your knowledge. For each problem, consider the potential edge cases and think about how you would make the code robust before revealing the solution.

This function is supposed to find the largest integer in an array. What could go wrong?

public int FindMax(int[] numbers) { int max = numbers[0]; for (int i = 1; i < numbers.Length; i++) { if (numbers[i] > max) { max = numbers[i]; } } return max; }

Edge Cases & Solution

  • Empty Array: The original code throws an `IndexOutOfRangeException`.
  • Null Array: The original code throws a `NullReferenceException`.

The solution is to check for these cases first and throw a meaningful exception.

public int FindMax(int[] numbers) { if (numbers == null || numbers.Length == 0) { throw new ArgumentException("Input array cannot be null or empty."); } int max = numbers[0]; for (int i = 1; i < numbers.Length; i++) { if (numbers[i] > max) { max = numbers[i]; } } return max; }

This function calculates the factorial of a number (n!). What are the hidden dangers?

public int Factorial(int n) { if (n == 0) return 1; return n * Factorial(n - 1); }

Edge Cases & Solution

  • Negative Input: The recursion will never end, causing a `StackOverflowException`.
  • Large Input: Factorials grow very fast. `13!` already exceeds the max value of an `int`, causing an overflow.

The solution validates the input and uses a `long` to handle larger results, preventing overflow for a wider range of inputs.

public long Factorial(int n) { if (n < 0) { throw new ArgumentException("Factorial is not defined for negative numbers."); } if (n > 20) // Factorial of 21 overflows even a long { throw new ArgumentException("Input too large for a long."); } if (n == 0) return 1; long result = 1; for (int i = 1; i <= n; i++) { result *= i; } return result; }

A simple function to get the first character. What could possibly fail?

public char GetFirstChar(string text) { return text[0]; }

Edge Cases & Solution

  • Null String: Accessing `text[0]` will throw a `NullReferenceException`.
  • Empty String: Accessing `text[0]` on an empty string will throw an `IndexOutOfRangeException`.

A robust solution checks for these conditions using `string.IsNullOrEmpty` before attempting to access the character.

public char GetFirstChar(string text) { if (string.IsNullOrEmpty(text)) { throw new ArgumentException("Input string cannot be null or empty."); } return text[0]; }

This function performs simple division. What's the most famous edge case in division?

public int Divide(int numerator, int denominator) { return numerator / denominator; }

Edge Cases & Solution

  • Division by Zero: If `denominator` is 0, the code will throw a `DivideByZeroException`.

The corrected code explicitly checks for a zero denominator to provide a more controlled error or response.

public int Divide(int numerator, int denominator) { if (denominator == 0) { throw new ArgumentException("Denominator cannot be zero."); } return numerator / denominator; }

Imagine `GetUser` returns a user object from a database, or `null` if not found. How can this code fail?

public class User { public string Name { get; set; } } public string GetUserName(int userId) { User user = GetUser(userId); // This might return null return user.Name; } // Assume this method exists and might return null private User GetUser(int id) { /* ... database logic ... */ return null; }

Edge Cases & Solution

  • Null Reference: If `GetUser` returns `null` because the user doesn't exist, calling `user.Name` will throw a `NullReferenceException`.

Always check for `null` after a database or API call that might not find a result.

public class User { public string Name { get; set; } } public string GetUserName(int userId) { User user = GetUser(userId); if (user == null) { return "User not found"; } return user.Name; } private User GetUser(int id) { return null; }
