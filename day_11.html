Day 11: Sliding Window (Variable Size)

The Variable-Size Sliding Window Technique

The variable-size sliding window technique is a powerful approach for solving problems that involve finding a subarray or substring that satisfies a certain condition. Unlike the fixed-size window, the size of this window (the distance between the start and end pointers) changes dynamically as we iterate through the data.

Core Idea

The general idea is to use two pointers, `start` and `end`, to define the current "window". We expand the window by moving the `end` pointer to the right. Once the window satisfies a certain condition, we try to shrink it from the left by moving the `start` pointer to the right, seeking a more optimal solution (e.g., a smaller subarray or a longer substring).

General Algorithm Template

  1. Initialize `start = 0`, `end = 0`, and other necessary variables.
  2. Iterate with the `end` pointer from left to right to expand the window.
  3. Update state based on the element at `end`.
  4. Enter a `while` loop to shrink the window from the left as long as a condition is met.
  5. Inside the `while` loop:
    • Update the answer (e.g., min length, max length).
    • Remove the effect of the element at `start`.
    • Increment `start`.

Problem 1: Smallest Subarray with Sum ≥ Target

Given an array of positive integers and a positive integer `target`, find the length of the smallest contiguous subarray whose sum is greater than or equal to `target`. If there is no such subarray, return 0.

C# Implementation & Explanation

We iterate through the array with the `end` pointer, adding each element to `windowSum`. Once `windowSum` is at least `target`, we enter a `while` loop. Inside this loop, we record the current window's length if it's the smallest we've seen, then we shrink the window by subtracting the `start` element's value and incrementing `start`. We repeat this shrinking process until the `windowSum` is less than the `target` again, at which point we continue expanding with the `end` pointer.

public class Solution {
    public int MinSubArrayLen(int target, int[] nums) {
        if (nums == null || nums.Length == 0) {
            return 0;
        }

        int minLength = int.MaxValue;
        int windowSum = 0;
        int windowStart = 0;

        for (int windowEnd = 0; windowEnd < nums.Length; windowEnd++) {
            windowSum += nums[windowEnd];

            while (windowSum >= target) {
                minLength = Math.Min(minLength, windowEnd - windowStart + 1);
                windowSum -= nums[windowStart];
                windowStart++;
            }
        }

        return minLength == int.MaxValue ? 0 : minLength;
    }
}

Problem 2: Longest Substring with K Unique Characters

Given a string, find the length of the longest substring that contains no more than `K` distinct characters.

C# Implementation & Explanation

We use a dictionary (hash map) to keep track of the frequency of characters in the current window. We expand the window with `end`, adding characters to the map. If the number of unique characters in the map exceeds `K`, we enter a `while` loop to shrink the window from the left. In the loop, we decrement the count of the `start` character. If its count becomes zero, we remove it from the map. We continue shrinking until the map's size is back to `K`. At each valid step (map size ≤ K), we update our `maxLength`.

public class Solution {
    public int LengthOfLongestSubstringKDistinct(string s, int k) {
        if (string.IsNullOrEmpty(s) || k == 0) {
            return 0;
        }

        int maxLength = 0;
        int windowStart = 0;
        Dictionary<char, int> charFrequency = new Dictionary<char, int>();

        for (int windowEnd = 0; windowEnd < s.Length; windowEnd++) {
            char rightChar = s[windowEnd];
            if (!charFrequency.ContainsKey(rightChar)) {
                charFrequency[rightChar] = 0;
            }
            charFrequency[rightChar]++;

            while (charFrequency.Count > k) {
                char leftChar = s[windowStart];
                charFrequency[leftChar]--;
                if (charFrequency[leftChar] == 0) {
                    charFrequency.Remove(leftChar);
                }
                windowStart++;
            }

            maxLength = Math.Max(maxLength, windowEnd - windowStart + 1);
        }

        return maxLength;
    }
}

Practice Problems

The best way to master a technique is to practice. Try applying the variable-size sliding window pattern to these problems. They offer slight variations that will challenge and solidify your understanding.

LeetCode 904. Fruit Into Baskets

This is essentially "Longest Substring with K=2 Unique Characters". A great starting point to reinforce the pattern.

LeetCode 3. Longest Substring Without Repeating Characters

Here, your condition for shrinking is finding a duplicate character within the window. The goal is to maximize length with all unique characters.

LeetCode 76. Minimum Window Substring

A more advanced version where you need to find the smallest window in one string that contains all characters of another string. You'll need two frequency maps for this.

LeetCode 424. Longest Repeating Character Replacement

This problem is tricky. The window's validity condition is `windowLength - maxFrequency <= k`. You shrink when this condition is violated.
