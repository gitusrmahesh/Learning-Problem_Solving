Longest Substring Without Repeating Characters

This section introduces a classic string manipulation problem: finding the length of the longest substring within a given string that does not contain any repeating characters. We will explore the highly efficient "Sliding Window" technique to solve this. The Sliding Window Approach

The sliding window is an algorithmic technique that involves maintaining a "window" (a sub-array or sub-string) over a larger data structure. For this problem, we use two pointers, `start` and `end`, to define the current window.

  1. Initialize a `start` pointer at the beginning of the string and an empty set to store characters in the current window.
  2. Iterate through the string with an `end` pointer.
  3. If the character at `end` is not in our set, add it. The window is valid. Update the maximum length found so far.
  4. If the character at `end` is already in the set, it means we have a repeat. We must shrink the window from the left by moving the `start` pointer forward and removing characters from the set until the repeating character is gone.
  5. Continue this process until the `end` pointer reaches the end of the string.

C# Implementation

public class Solution {
    public int LengthOfLongestSubstring(string s) {
        if (string.IsNullOrEmpty(s)) return 0;

        int maxLength = 0;
        int start = 0;
        var charSet = new HashSet<char>();

        for (int end = 0; end < s.Length; end++) {
            // While the set contains the current character,
            // shrink the window from the left.
            while (charSet.Contains(s[end])) {
                charSet.Remove(s[start]);
                start++;
            }
            
            // Add the current character to the set
            charSet.Add(s[end]);
            
            // Update the max length
            maxLength = Math.Max(maxLength, end - start + 1);
        }

        return maxLength;
    }
}

Complexity Analysis

Time Complexity: O(N)

Although there is a nested `while` loop, each character in the string is visited by the `start` and `end` pointers at most once. Therefore, the complexity is linear.

Space Complexity: O(k)

Where `k` is the size of the character set (e.g., 26 for lowercase English letters, 128 for ASCII). The space used by the `HashSet` is bounded by the number of unique characters.

Longest Common Subsequence (LCS)

This section delves into the Longest Common Subsequence problem, a fundamental concept in computer science with applications in bioinformatics (DNA sequencing) and data comparison (diff utilities). Unlike a substring, a subsequence's characters do not need to be contiguous. We will solve this using Dynamic Programming, building a table to store solutions to subproblems. The Dynamic Programming Approach

We use a 2D array, let's call it `dp`, where `dp[i][j]` stores the length of the LCS for the first `i` characters of `string1` and the first `j` characters of `string2`.

  1. Create a `dp` table of size `(m+1) x (n+1)`, where `m` and `n` are the lengths of the two strings. Initialize it with zeros.
  2. Iterate through the strings from `i=1` to `m` and `j=1` to `n`.
  3. If `string1[i-1]` equals `string2[j-1]`, the characters match! The LCS length is one more than the LCS of the strings without these characters. So, `dp[i][j] = 1 + dp[i-1][j-1]`.
  4. If the characters do not match, the LCS length is the maximum of the LCS found by excluding one character from either string. So, `dp[i][j] = max(dp[i-1][j], dp[i][j-1])`.
  5. The value at `dp[m][n]` will be the length of the LCS. We can then backtrack from this cell to reconstruct the actual subsequence.

C# Implementation

public class Solution {
    public int LongestCommonSubsequence(string text1, string text2) {
        int m = text1.Length;
        int n = text2.Length;
        
        // dp[i, j] will be the length of LCS of text1[0..i-1] and text2[0..j-1]
        int[,] dp = new int[m + 1, n + 1];
        
        for (int i = 1; i <= m; i++) {
            for (int j = 1; j <= n; j++) {
                if (text1[i - 1] == text2[j - 1]) {
                    // If characters match, add 1 to the result from the previous diagonal cell
                    dp[i, j] = 1 + dp[i - 1, j - 1];
                } else {
                    // If characters don't match, take the max from the top or left cell
                    dp[i, j] = Math.Max(dp[i - 1, j], dp[i, j - 1]);
                }
            }
        }
        
        // The bottom-right cell contains the length of the LCS
        return dp[m, n];
    }
}
// Note: This code finds the length. Reconstructing the string requires backtracking.

Complexity Analysis

Time Complexity: O(m * n)

We iterate through a 2D grid of size `m x n`, where `m` and `n` are the lengths of the two strings. Each cell computation takes constant time.

Space Complexity: O(m * n)

We use a 2D `dp` table of size `(m+1) x (n+1)` to store the intermediate results.
