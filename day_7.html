Understanding Complexity

Complexity analysis helps us predict how an algorithm's resource usage (like time or memory) scales with the size of the input. Choosing an efficient algorithm is often the most critical step in problem-solving. This section allows you to compare two different approaches to the same problem and see how their complexities differ dramatically.

Approach 1: Brute Force O(n²)

public int[] TwoSum_BruteForce(int[] nums, int target) {
  for (int i = 0; i < nums.Length; i++) {
    for (int j = i + 1; j < nums.Length; j++) {
      if (nums[i] + nums[j] == target) {
        return new int[] { i, j };
      }
    }
  }
  return null;
}

Approach 2: Optimized O(n)

public int[] TwoSum_Optimized(int[] nums, int target) {
  var map = new Dictionary<int, int>();
  for (int i = 0; i < nums.Length; i++) {
    int complement = target - nums[i];
    if (map.ContainsKey(complement)) {
      return new int[] { map[complement], i };
    }
    map[nums[i]] = i;
  }
  return null;
}

The Art of the Dry Run

A dry run is the process of manually tracing your code's execution with a specific input, step by step. This is an invaluable skill for verifying your logic before you even write code and for finding subtle bugs that are hard to spot. Use the simulator below to walk through an iterative function and observe how variables change at each step.

C# Function: Iterative Factorial

public int Factorial(int n) {
  int result = 1;
  for (int i = 1; i <= n; i++) {
    result = result * i;
  }
  return result;
}
