Prefix Sum

Create a new array where `prefix[i]` is the sum of all elements from `nums[0]` to `nums[i]`. Allows calculating any subarray sum in O(1) time. Ideal for frequent subarray sum queries.

public int[] BuildPrefixSum(int[] nums) {
    int n = nums.Length,[] prefix = new int[n];
    prefix[0] = nums[0];
    for (int i=1; i<n; i++) prefix[i]=prefix[i-1]+nums[i];
    return prefix;
}

Sliding Window

Efficiently finds a subarray satisfying a condition by maintaining a "window" and sliding it through the array. Reduces time complexity from O(n²) to O(n) by avoiding redundant calculations.

public int FindMaxSum(int[] nums, int k) {
    int maxSum=0, windowSum=0;
    for(int i=0; i<k; i++) windowSum += nums[i];
    for(int i=k; i<nums.Length; i++) {
        windowSum += nums[i] - nums[i-k];
        maxSum = Math.Max(maxSum, windowSum);
    } return maxSum;
}

Two Pointers

Uses two pointers to iterate through an array, often from different directions on sorted data. Narrows the search space efficiently, providing O(n) solutions for problems that might otherwise take O(n²).

public bool HasPairWithSum(int[] sorted, int target) {
    int L=0, R=sorted.Length-1;
    while (L < R) {
        int sum = sorted[L] + sorted[R];
        if (sum == target) return true;
        if (sum < target) L++; else R--;
    } return false;
}

Contest Summary

Congratulations on completing the contest. This session highlighted how different array problems can be efficiently solved by recognizing the underlying pattern. Let's review the core principles:

  • ▸
    Fixed-Size Sliding Window: Ideal for "max/min of subarray size k" problems. The key is constant-time window updates: add the new element, subtract the old.
  • ▸
    Variable-Size Sliding Window: Perfect for finding subarrays that meet a target value. The window expands and contracts dynamically, ensuring linear time complexity.
  • ▸
    Two Pointers on Sorted Arrays: Unbeatable for finding pairs that sum to a target. By moving pointers from opposite ends, you intelligently eliminate half the search space in each step.
  • ▸
    Prefix Sums with Hashing: The go-to for complex subarray sum problems, especially those involving divisibility. It transforms the problem into finding pairs of indices with specific prefix sum properties.
