Day 22: C# String Basics

This section covers the fundamental C# concepts for string manipulation, focusing on reversal and palindrome detection. Understanding these techniques is crucial for solving a wide range of string-based problems. We'll explore several common methods and discuss their underlying mechanics.

1. Understanding String Immutability in C#

A core concept in C# is that string objects are immutable. This means that once a string is created, its value cannot be changed. Any method that appears to modify a string, like reversing it, actually creates and returns a new string object. This has important performance implications, especially when dealing with many modifications in a loop.

2. Reversing a String

There are multiple ways to reverse a string in C#. Let's look at three popular approaches.

Method A: Using Array.Reverse()

This is often the most readable and straightforward method. You convert the string to a character array, reverse the array in-place, and then create a new string from the reversed array.

public static string ReverseStringWithArray(string text)
{
    if (string.IsNullOrEmpty(text)) return text;

    char[] charArray = text.ToCharArray();
    Array.Reverse(charArray);
    return new string(charArray);
}

Method B: Using a Loop

This approach builds the reversed string character by character, iterating from the end of the original string to the beginning. This method avoids creating an intermediate character array but can be less efficient due to repeated string concatenations (which create new string objects in each iteration).

public static string ReverseStringWithLoop(string text)
{
    if (string.IsNullOrEmpty(text)) return text;
    
    // Using StringBuilder is much more efficient than string concatenation
    StringBuilder reversedString = new StringBuilder();
    for (int i = text.Length - 1; i >= 0; i--)
    {
        reversedString.Append(text[i]);
    }
    return reversedString.ToString();
}

3. Checking for a Palindrome

A palindrome is a word, phrase, or sequence that reads the same backward as forward (e.g., "madam", "level"). The easiest way to check for a palindrome is to reverse the string and see if it equals the original. It's important to handle casing and non-alphanumeric characters based on the problem's requirements.

Simple Palindrome Check

This version compares a string with its reversed version. For a robust check, you should typically convert the string to a common case (e.g., lowercase).

public static bool IsPalindromeSimple(string text)
{
    if (string.IsNullOrEmpty(text)) return true;
    
    string normalized = text.ToLower();
    string reversed = ReverseStringWithArray(normalized); // Re-using our reverse method
    
    return normalized == reversed;
}

Optimized Two-Pointer Approach

A more memory-efficient method is to use two pointers. One starts at the beginning of the string and one at the end. You compare the characters at both pointers, moving them towards the center. If you find a mismatch, it's not a palindrome. This avoids creating a new reversed string.

public static bool IsPalindromeTwoPointers(string text)
{
    if (string.IsNullOrEmpty(text)) return true;
    
    string normalized = text.ToLower();
    int left = 0;
    int right = normalized.Length - 1;
    
    while (left < right)
    {
        if (normalized[left] != normalized[right])
        {
            return false;
        }
        left++;
        right--;
    }
    
    return true;
}
