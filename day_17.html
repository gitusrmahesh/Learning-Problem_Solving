Kth Largest Element

This section explores efficient ways to find the Kth largest element in an unsorted array. This is a classic problem that highlights how choosing the right data structure can dramatically improve performance over naive sorting. We will focus on a heap-based approach, which provides a good balance of time complexity and implementation simplicity.

The Challenge

Given an integer array `nums` and an integer `k`, return the `k`th largest element in the array. Note that it is the `k`th largest element in the sorted order, not the `k`th distinct element.

Optimal Approach: Min-Heap

A highly efficient method uses a Min-Heap (in C#, this can be implemented with a `PriorityQueue`). The idea is to maintain a heap of size `k`.

  • Iterate through the array elements.
  • For each element, add it to the Min-Heap.
  • If the heap's size exceeds `k`, remove the smallest element (the root of the Min-Heap).
  • After iterating through all elements, the root of the heap will be the Kth largest element.

This approach has a time complexity of O(N log K), which is much better than sorting the entire array (O(N log N)) when `k` is small compared to `N`.

C# Implementation 

public class Solution {
    public int FindKthLargest(int[] nums, int k) {
        // C# 6.0 introduced PriorityQueue
        // The first argument is the item, the second is its priority.
        // By default it's a Min-Heap, so we use the number itself as priority.
        var minHeap = new PriorityQueue<int, int>();

        foreach (int num in nums) {
            minHeap.Enqueue(num, num);
            if (minHeap.Count > k) {
                minHeap.Dequeue();
            }
        }

        // The smallest element in a heap of size k is the kth largest overall.
        return minHeap.Peek();
    }
}

Sort Array by Increasing Frequency

This problem requires sorting an array based on the frequency of its elements. If multiple elements share the same frequency, they must be sorted in descending order. This task combines counting, custom sorting logic, and data manipulation, making it an excellent exercise for strengthening your problem-solving skills with dictionaries and LINQ in C#.

The Challenge

Given an array of integers `nums`, sort the array in increasing order based on the frequency of the values. If multiple values have the same frequency, sort them in decreasing order.

Approach: Count and Custom Sort

A clear and effective strategy involves two main steps:

  • Count Frequencies: Use a Dictionary (or a HashMap) to iterate through the array and store the count of each number.
  • Custom Sort: Use a custom sorting logic. In C#, LINQ's `OrderBy` and `ThenByDescending` methods are perfect for this. First, order the numbers by their frequency (from the dictionary) in ascending order. Then, for numbers with the same frequency, order them by their value in descending order.

C# Implementation (using LINQ) 

using System.Collections.Generic;
using System.Linq;

public class Solution {
    public int[] FrequencySort(int[] nums) {
        // 1. Count frequencies of each number
        var frequencyMap = new Dictionary<int, int>();
        foreach (int num in nums) {
            if (!frequencyMap.ContainsKey(num)) {
                frequencyMap[num] = 0;
            }
            frequencyMap[num]++;
        }

        // 2. Sort the original array using custom logic
        // We use Array.Sort with a custom Comparison delegate
        System.Array.Sort(nums, (a, b) => {
            int freqA = frequencyMap[a];
            int freqB = frequencyMap[b];

            if (freqA != freqB) {
                // Sort by frequency in increasing order
                return freqA.CompareTo(freqB);
            } else {
                // If frequencies are the same, sort by value in decreasing order
                return b.CompareTo(a);
            }
        });

        return nums;
    }
}
