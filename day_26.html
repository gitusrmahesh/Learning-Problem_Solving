An Introduction to String Hashing

String hashing is a powerful technique used to convert a string of any length into a fixed-size integer, known as a "hash" or "hash code." This hash acts as a unique fingerprint for the string, enabling incredibly fast comparisons and lookups. Instead of comparing two long strings character by character (an O(n) operation, where n is the length of the string), we can compare their integer hashes (an O(1) operation). This is the core principle behind data structures like hash maps, and it has wide-ranging applications in areas like database indexing, caching, and detecting duplicate data.

Core Idea: The goal of a good hash function is to distribute a set of strings as evenly as possible into a fixed number of "buckets," minimizing the chances of two different strings producing the same hash. When two different strings do produce the same hash, it's called a collision. Modern hashing algorithms are designed to make collisions extremely rare.

A Simple Hashing Example (Conceptual)

To understand the basic concept, let's imagine a very simple (and not very good) hash function that just sums the ASCII values of the characters in a string. Let's see how it would hash the word "cat":

  • 'c' has an ASCII value of 99
  • 'a' has an ASCII value of 97
  • 't' has an ASCII value of 116

Hash("cat") = 99 + 97 + 116 = 312.

Now, if we wanted to find "cat" in a data structure, we could instantly jump to bucket #312 instead of searching linearly. However, this simple function has a major flaw: "act" would also hash to 312, causing a collision. Real-world algorithms, like the polynomial rolling hash, are far more sophisticated to avoid such simple collisions.

Hashmaps for Strings in C#

In C#, the most common implementation of a hash map is the `Dictionary` class. When you use a `string` as a key, C# internally calls the string's `GetHashCode()` method to compute its hash. This hash code is then used to determine which "bucket" to store the associated value in. This process makes adding, removing, and retrieving items incredibly efficient. On average, these operations take constant time, O(1). In the rare case of many hash collisions, the performance can degrade to linear time, O(n), but this is uncommon with well-designed hash functions.

using System;
using System.Collections.Generic;

public class Program
{
    public static void Main()
    {
        // 1. Initialize a Dictionary with string keys
        Dictionary<string, int> fruitPrices = new Dictionary<string, int>();

        // 2. Add elements
        fruitPrices["apple"] = 100; // Price in cents
        fruitPrices["orange"] = 80;
        fruitPrices["banana"] = 50;

        // 3. Access an element
        Console.WriteLine($"The price of an apple is {fruitPrices["apple"]} cents.");

        // 4. Check if a key exists
        if (fruitPrices.ContainsKey("banana"))
        {
            Console.WriteLine("We have pricing for bananas.");
        }
    }
}

Application: Frequency Counting

One of the most common and useful applications of hashmaps is frequency counting: tallying the occurrences of items in a collection. For strings, this often means counting how many times each character appears. A hashmap is perfect for this task due to its O(1) average time complexity for lookups and insertions. The algorithm is straightforward: iterate through the string's characters. For each character, use it as a key in the map. If the character isn't in the map yet, add it with a count of 1. If it's already there, simply retrieve the current count, increment it, and update the value. This demonstrates a classic space-time tradeoff: we use extra memory (for the hashmap) to achieve a significant gain in processing speed. Try it below by typing any text to see a live chart of its character frequencies.

Example: Find the Most Frequent Character

public class FrequencyExample {
    public char FindMostFrequentChar(string input) {
        if (string.IsNullOrEmpty(input)) {
            throw new ArgumentException("Input string cannot be null or empty.");
        }

        Dictionary<char, int> counts = new Dictionary<char, int>();
        foreach (char c in input) {
            if (counts.ContainsKey(c)) {
                counts[c]++;
            } else {
                counts[c] = 1;
            }
        }

        char mostFrequent = ' ';
        int maxCount = 0;
        foreach (KeyValuePair<char, int> pair in counts) {
            if (pair.Value > maxCount) {
                maxCount = pair.Value;
                mostFrequent = pair.Key;
            }
        }
        return mostFrequent;
    }
}
// Logic: First, we build a frequency map of all characters, just like in the basic frequency counting pattern.
// Then, we iterate through the key-value pairs in our completed map to find the entry with the highest value (count).

Practice Problems

The best way to solidify your understanding is to solve problems. Here are some intermediate-level challenges that can be efficiently solved using hashmaps. Try to solve them on your own before revealing the C# solution.

Given two strings `s` and `t`, return `true` if `t` is an anagram of `s`, and `false` otherwise. An anagram is a word or phrase formed by rearranging the letters of a different word, using all original letters exactly once.

C# Solution:

public class Solution {
    public bool IsAnagram(string s, string t) {
        if (s.Length != t.Length) return false;

        Dictionary<char, int> sCounts = new Dictionary<char, int>();
        
        foreach (char c in s) {
            if (sCounts.ContainsKey(c)) {
                sCounts[c]++;
            } else {
                sCounts[c] = 1;
            }
        }
        
        foreach (char c in t) {
            if (!sCounts.ContainsKey(c) || sCounts[c] == 0) {
                return false;
            }
            sCounts[c]--;
        }
        
        return true;
    }
}
// Logic: Anagrams must have the same characters with the same frequencies.
// 1. First, a quick check: if the lengths are different, they can't be anagrams.
// 2. Create a frequency map for the first string 's'.
// 3. Iterate through the second string 't'. For each character in 't', decrement its count in the map.
// 4. If a character from 't' is not found in the map, or its count is already zero, they are not anagrams.
// 5. If the loop completes, it means every character in 't' was accounted for in 's', so they are anagrams.

Given an array of strings `strs`, group the anagrams together. You can return the answer in any order.

C# Solution:

public class Solution {
    public IList<IList<string>> GroupAnagrams(string[] strs) {
        if (strs == null || strs.Length == 0) return new List<IList<string>>();

        Dictionary<string, List<string>> map = new Dictionary<string, List<string>>();

        foreach (string s in strs) {
            char[] charArray = s.ToCharArray();
            Array.Sort(charArray);
            string key = new string(charArray);
            
            if (!map.ContainsKey(key)) {
                map[key] = new List<string>();
            }
            map[key].Add(s);
        }

        return new List<IList<string>>(map.Values);
    }
}
// Logic: The key insight is that all anagrams become the same string when their characters are sorted alphabetically.
// This sorted string can serve as a "canonical representation" or a unique key for any group of anagrams.
// 1. We create a dictionary where the key is the sorted string, and the value is a list of original strings.
// 2. We loop through each input string.
// 3. For each string, we sort its characters to create the key.
// 4. We use this key to add the original string to the correct list in our dictionary.
// 5. Finally, the values of the dictionary are the grouped anagrams.

Given a string `s`, find the first non-repeating character in it and return its index. If it does not exist, return -1.

C# Solution:

public class Solution {
    public int FirstUniqChar(string s) {
        Dictionary<char, int> counts = new Dictionary<char, int>();

        // First pass: build frequency map
        foreach (char c in s) {
            if (counts.ContainsKey(c)) {
                counts[c]++;
            } else {
                counts[c] = 1;
            }
        }

        // Second pass: find the first character with a count of 1
        for (int i = 0; i < s.Length; i++) {
            if (counts[s[i]] == 1) {
                return i;
            }
        }

        return -1;
    }
}
// Logic: This problem requires preserving the original order to find the *first* unique character.
// A single hash map doesn't store order, so a two-pass approach is effective.
// 1. The first pass populates a hashmap with character frequencies. This tells us *which* characters are unique, but not their original positions.
// 2. The second pass iterates through the *original string* from the beginning.
// 3. For each character, we look up its count in the map. The first time we find a character with a count of 1, we return its current index.
// 4. If we finish the loop without finding a unique character, we return -1.

Given two strings `ransomNote` and `magazine`, return `true` if `ransomNote` can be constructed by using the letters from `magazine` and `false` otherwise. Each letter in `magazine` can only be used once in `ransomNote`.

C# Solution:

public class Solution {
    public bool CanConstruct(string ransomNote, string magazine) {
        Dictionary<char, int> magCounts = new Dictionary<char, int>();

        // Count all characters in the magazine
        foreach (char c in magazine) {
            if (magCounts.ContainsKey(c)) {
                magCounts[c]++;
            } else {
                magCounts[c] = 1;
            }
        }

        // Check if there are enough characters for the ransom note
        foreach (char c in ransomNote) {
            if (!magCounts.ContainsKey(c) || magCounts[c] == 0) {
                return false; // Not enough of this character
            }
            magCounts[c]--; // "Use" one character
        }

        return true;
    }
}
// Logic: We need to ensure that every character required for the ransom note is available in the magazine in sufficient quantity.
// 1. We first create a frequency map of all available characters from the 'magazine'. This is our inventory.
// 2. Then, we iterate through the 'ransomNote'. For each character needed, we check our inventory.
// 3. If the character is not in our map, or its count in the map is zero, we don't have it, so we immediately return false.
// 4. If we do have the character, we decrement its count in the map to signify that we've "used" it.
// 5. If we successfully complete the loop over the ransom note, it means all characters were available, and we return true.
