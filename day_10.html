The Problem: Max Sum of a Subarray of Size K

Given an array of integers and a number 'k', our goal is to find the maximum sum of any contiguous subarray of size 'k'. A "subarray" is a sequence of connected elements from the array. For example, in the array `[2, 1, 5, 1, 3, 2]`, if k=3, one possible subarray is `[1, 5, 1]`, and its sum is 7. We need to find the subarray of size 3 that has the largest possible sum.

Approach 1: The Brute-Force Method

The most straightforward way to solve this is to consider every possible subarray of size 'k'. We can use a loop to determine the starting point of our subarray, and then an inner loop to sum up the next 'k' elements. We would keep track of the maximum sum found so far and update it whenever we find a larger sum. 

// C# Brute-Force Example
public int FindMaxSumSubarray_BruteForce(int[] arr, int k) {
    int maxSum = 0;
    for (int i = 0; i <= arr.Length - k; i++) {
        int windowSum = 0;
        for (int j = i; j < i + k; j++) {
            windowSum += arr[j];
        }
        maxSum = Math.Max(maxSum, windowSum);
    }
    return maxSum;
}

While this works, it's inefficient. Notice that for each step, we are recalculating the sum of elements that were already part of the previous subarray. This repeated work becomes very costly as the array size grows.

Approach 2: The Sliding Window Technique

A much more efficient approach is the "Sliding Window". Instead of recalculating the sum for each subarray, we maintain a "window" of size 'k'. As we move from one subarray to the next, we simply subtract the element that is leaving the window and add the new element that is entering it. This avoids redundant calculations. C# Implementation

The code for the sliding window approach is much cleaner and more efficient. We initialize the window sum with the first 'k' elements. Then, we iterate through the rest of the array, sliding the window one element at a time by subtracting the outgoing element and adding the incoming one. 

// C# Sliding Window Example
public int FindMaxSumSubarray(int[] arr, int k) {
    if (arr.Length < k) {
        return 0; // Or throw an exception
    }

    int windowSum = 0;
    int maxSum = 0;
    int windowStart = 0;

    for (int windowEnd = 0; windowEnd < arr.Length; windowEnd++) {
        windowSum += arr[windowEnd]; // Add the next element

        // Slide the window when we hit the size 'k'
        if (windowEnd >= k - 1) {
            maxSum = Math.Max(maxSum, windowSum); // Update max sum
            windowSum -= arr[windowStart]; // Subtract the element leaving
            windowStart++; // Slide window forward
        }
    }
    return maxSum;
}

Complexity Analysis

The performance difference between the two approaches is significant. The sliding window technique avoids the nested loop, allowing us to process the array in a single pass.

Brute-Force

O(N * K)

Time complexity is high because for each of the N-K+1 windows, we perform K additions.

Sliding Window

O(N)

Time complexity is linear because we iterate through the array only once.

Practice Problems

This technique is fundamental for solving many array and string-based problems. Here are a few classic examples to practice:

  • Find the average of all contiguous subarrays of size 'K'.
  • Find the smallest subarray with a given sum (this is a variable-size window, a topic for another day!).
  • Given a string, find the length of the longest substring with no more than 'K' distinct characters.
  • Find all anagrams of a pattern in a given string.
  • Maximum number of vowels in a substring of given length.
