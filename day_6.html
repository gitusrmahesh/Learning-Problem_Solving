Easy Array: Two Number Sum

This section introduces a classic array problem. The goal is to find if any two numbers in a given array sum up to a specific target value. This type of problem tests your ability to efficiently iterate through data structures and manage state. We will explore a solution using a hash set for optimal performance.

Problem Statement:

Write a function that takes in a non-empty array of distinct integers and an integer representing a target sum. If any two numbers in the input array sum up to the target sum, the function should return them in an array. If no two numbers sum up to the target sum, the function should return an empty array.

C# Solution 
using System;
using System.Collections.Generic;

public class Program {
    // O(n) time | O(n) space
    public static int[] TwoNumberSum(int[] array, int targetSum) {
        HashSet<int> nums = new HashSet<int>();
        foreach (int num in array) {
            int potentialMatch = targetSum - num;
            if (nums.Contains(potentialMatch)) {
                return new int[] {potentialMatch, num};
            } else {
                nums.Add(num);
            }
        }
        return new int[0];
    }
}

Explanation:

The code iterates through the array once. For each number `num`, it calculates the `potentialMatch` needed to reach the `targetSum`. It checks a `HashSet` to see if this `potentialMatch` has been seen before. A HashSet provides O(1) average time complexity for lookups. If the match exists, we've found our pair. If not, the current `num` is added to the set for future checks. This avoids a nested loop (which would be O(n²)) and gives us a time-efficient O(n) solution.

Easy Math: Validate Subsequence

This problem focuses on sequence validation, a common task that falls under the umbrella of both array and mathematical logic problems. The challenge is to determine if one sequence of numbers appears within another, maintaining their relative order. The solution demonstrates the power of using pointers to track progress through collections.

Problem Statement:

Given two non-empty arrays of integers, write a function that determines whether the second array is a subsequence of the first one. A subsequence is a set of numbers that aren't necessarily adjacent in the array but are in the same order as they appear in the array.

C# Solution 
using System;
using System.Collections.Generic;

public class Program {
    // O(n) time | O(1) space - where n is the length of the main array
    public static bool IsValidSubsequence(List<int> array, List<int> sequence) {
        int seqIdx = 0;
        foreach (int value in array) {
            if (seqIdx == sequence.Count) {
                break;
            }
            if (sequence[seqIdx] == value) {
                seqIdx++;
            }
        }
        return seqIdx == sequence.Count;
    }
}

Explanation:

We use a single pointer, `seqIdx`, to keep track of our position in the `sequence` array. We iterate through the main `array`. If the current element in `array` matches the element at `sequence[seqIdx]`, it means we've found the next number in our subsequence, so we increment `seqIdx`. We continue this until we've either exhausted the main `array` or found all elements in the `sequence`. The function returns true only if `seqIdx` has reached the length of the `sequence`, indicating all its elements were found in order.

Complexity Reasoning

Understanding time and space complexity (Big O notation) is crucial. It's not about writing code, but analyzing it to predict its performance as input size grows. This skill helps you choose the right algorithm and avoid performance bottlenecks. Below, analyze the code snippet and determine its time complexity.

Task:

Determine the time complexity of the following C# function, which finds the first duplicate value in an array.

public int FirstDuplicateValue(int[] array) {
    for (int i = 0; i < array.Length; i++) {
        for (int j = i + 1; j < array.Length; j++) {
            if (array[i] == array[j]) {
                return array[i];
            }
        }
    }
    return -1;
}

What is the time complexity?

Correct! The answer is O(n²).

The function uses nested loops. The outer loop runs approximately `n` times (where `n` is the length of the array). For each iteration of the outer loop, the inner loop also runs, on average, a proportional number of times. This results in roughly `n * n` comparisons in the worst case, giving it a quadratic time complexity of O(n²).
