What is Bit Manipulation?

Bit manipulation involves performing operations on numbers at the level of their individual bits. Processors have native instructions for bitwise operations, making them extremely fast. While modern compilers and interpreters are very good at optimization, direct bit manipulation can still be a powerful tool for solving certain problems with remarkable efficiency in terms of both time and memory. This lesson explores common, practical applications you might encounter in technical interviews and competitive programming.

1. The Single Number Problem

This section introduces a classic problem: in an array where every element appears twice except for one, find that single element. The XOR (exclusive OR) bitwise operator provides an elegant and highly efficient solution. By understanding XOR's properties—that any number XORed with itself is zero, and any number XORed with zero is itself—we can find the unique number by XORing all elements together. The paired numbers will cancel out, leaving only the single number.

C# Implementation

public class Solution {
    public int SingleNumber(int[] nums) {
        int result = 0;
        foreach (int num in nums) {
            result ^= num;
        }
        return result;
    }
}

2. Power of Two Check

Determining if a number is a power of two is a common requirement. A number is a power of two if it has exactly one '1' bit in its binary representation (e.g., 8 is `1000`). This section demonstrates a clever trick using the AND bitwise operator. For any power of two `n`, `n-1` will have all bits to the right of that single '1' bit set to '1'. Therefore, `n & (n - 1)` will always be zero. This provides a constant-time check.

C# Implementation

public class Solution {
    public bool IsPowerOfTwo(int n) {
        // A power of two must be positive and
        // have only one bit set to '1'.
        return n > 0 && (n & (n - 1)) == 0;
    }
}

3. Swap Numbers Using XOR

While mostly an academic puzzle in modern high-level languages, the XOR swap algorithm is a classic demonstration of bit manipulation's power. It allows you to swap the values of two integer variables without using a temporary third variable. This technique relies on the properties of XOR to store and retrieve the original values within the two variables themselves. This section breaks down the three-step process to make it clear how the swap occurs at the bit level.

C# Implementation

public class Solution {
    public void Swap(ref int a, ref int b) {
        if (a == b) return; // Important edge case
        
        a = a ^ b;
        b = b ^ a; // Now b is original a's value
        a = a ^ b; // Now a is original b's value
    }
}

4. Practice Problems

The best way to master bit manipulation is to practice. Here are five problems that build upon the concepts covered in this guide. They will challenge you to apply bitwise operators in different contexts.

1. Number of 1 Bits (Hamming Weight)

Write a function that takes the binary representation of an unsigned integer and returns the number of '1' bits it has.

public class Solution {
    public int HammingWeight(uint n) {
        int count = 0;
        while (n != 0) {
            n &= (n - 1);
            count++;
        }
        return count;
    }
}

2. Missing Number

Given an array `nums` containing `n` distinct numbers in the range `[0, n]`, return the only number in the range that is missing from the array.

public class Solution {
    public int MissingNumber(int[] nums) {
        int missing = nums.Length;
        for (int i = 0; i < nums.Length; i++) {
            missing ^= i ^ nums[i];
        }
        return missing;
    }
}

3. Reverse Bits

Reverse bits of a given 32-bit unsigned integer.

public class Solution {
    public uint ReverseBits(uint n) {
        uint reversed = 0;
        for (int i = 0; i < 32; i++) {
            reversed = (reversed << 1) | (n & 1);
            n >>= 1;
        }
        return reversed;
    }
}

4. Subsets (Power Set)

Given an integer array `nums` of unique elements, return all possible subsets. The solution set must not contain duplicate subsets.

public class Solution {
    public IList<IList<int>> Subsets(int[] nums) {
        var subsets = new List<IList<int>>();
        int n = nums.Length;
        int max = 1 << n;

        for (int i = 0; i < max; i++) {
            var currentSubset = new List<int>();
            for (int j = 0; j < n; j++) {
                if ((i & (1 << j)) > 0) {
                    currentSubset.Add(nums[j]);
                }
            }
            subsets.Add(currentSubset);
        }
        return subsets;
    }
}

5. Single Number II

Given an integer array `nums` where every element appears three times except for one, which appears exactly once. Find the single element.

public class Solution {
    public int SingleNumber(int[] nums) {
        int result = 0;
        for (int i = 0; i < 32; i++) {
            int sum = 0;
            foreach (int num in nums) {
                sum += (num >> i) & 1;
            }
            if (sum % 3 != 0) {
                result |= (1 << i);
            }
        }
        return result;
    }
}
