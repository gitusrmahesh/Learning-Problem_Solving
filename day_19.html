Counting Set Bits

Brian Kernighan’s Algorithm

This clever algorithm provides an efficient way to count the number of '1's (set bits) in a number's binary representation. Its efficiency comes from a simple yet powerful trick: the operation n & (n - 1) unsets (turns off) the rightmost set bit of n. By repeating this operation until the number becomes zero, we can count exactly how many set bits were originally present.

The number of iterations is equal to the number of set bits, making it much faster than checking every single bit in the number, especially for sparse numbers (numbers with few set bits). Use the visualizer to see this process in action. 

public static int CountSetBits(int n)
{
    int count = 0;
    while (n > 0)
    {
        // This operation clears the rightmost set bit
        n &= (n - 1);
        count++;
    }
    return count;
}

Subset Generation

Using Bitmasks

Bit manipulation offers an elegant way to generate all possible subsets (the power set) of a given set. If a set has n elements, there are 2^n possible subsets. We can map each subset to a number from 0 to 2^n - 1.

The binary representation of each number acts as a "mask". If the i-th bit in the number is 1, we include the i-th element of the set in the current subset. By iterating through all numbers in the range, we can generate every combination. 

public static List<List<T>> GenerateSubsets<T>(T[] set)
{
    var allSubsets = new List<List<T>>();
    int n = set.Length;
    int max = 1 << n; // Equivalent to 2^n

    for (int i = 0; i < max; i++)
    {
        var subset = new List<T>();
        for (int j = 0; j < n; j++)
        {
            // Check if the j-th bit is set in i
            if ((i & (1 << j)) > 0)
            {
                subset.Add(set[j]);
            }
        }
        allSubsets.Add(subset);
    }
    return allSubsets;
}
