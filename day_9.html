Day 9: Prefix & Suffix Arrays

Introduction: The Power of Pre-computation

Welcome to Day 9! Today, we'll explore a fundamental and incredibly powerful technique in array manipulation: **Prefix and Suffix Sums**. Many problems involve calculating the sum of elements within a specific range of an array, often multiple times. A naive approach of iterating through the range for each query can be very slow.

Prefix and Suffix arrays are a form of pre-computation. By investing a little time upfront to create an auxiliary array, we can answer subsequent queries about sums in constant time—a massive performance improvement. This concept is a cornerstone for solving many "Subarray Sum" problems efficiently.

1. The Prefix Sum Array

A prefix sum array, `prefix`, is an array where `prefix[i]` stores the cumulative sum of all elements from the beginning of the original array, `arr`, up to index `i`.

prefix[i] = arr[0] + arr[1] + ... + arr[i]

It can be calculated iteratively: `prefix[i] = prefix[i-1] + arr[i]`, with the base case `prefix[0] = arr[0]`. This allows us to build the entire prefix sum array in O(n) time, where n is the size of the original array.

C# Implementation

public int[] BuildPrefixSum(int[] nums) { if (nums == null || nums.Length == 0) { return new int[0]; } int n = nums.Length; int[] prefixSum = new int[n]; prefixSum[0] = nums[0]; for (int i = 1; i < n; i++) { prefixSum[i] = prefixSum[i-1] + nums[i]; } return prefixSum; }

2. The Suffix Sum Array

Similarly, a suffix sum array, `suffix`, is an array where `suffix[i]` stores the cumulative sum of all elements from index `i` to the end of the original array, `arr`.

suffix[i] = arr[i] + arr[i+1] + ... + arr[n-1]

It's calculated by iterating from the end of the array: `suffix[i] = suffix[i+1] + arr[i]`, with the base case `suffix[n-1] = arr[n-1]`. This also takes O(n) time.

C# Implementation

public int[] BuildSuffixSum(int[] nums) { if (nums == null || nums.Length == 0) { return new int[0]; } int n = nums.Length; int[] suffixSum = new int[n]; suffixSum[n-1] = nums[n-1]; for (int i = n - 2; i >= 0; i--) { suffixSum[i] = suffixSum[i+1] + nums[i]; } return suffixSum; }

3. Application: Range Sum Queries

This is the classic application of prefix sums. The problem is to find the sum of elements in a given range `[L, R]` (inclusive) of an array. With a pre-computed prefix sum array, we can answer any such query in O(1) time.

The sum of elements from index `L` to `R` is the total sum up to `R` minus the total sum up to `L-1`.

Sum(L, R) = prefix[R] - prefix[L-1]

An important edge case is when `L = 0`. In this case, the sum is simply `prefix[R]`, as there are no preceding elements to subtract.

C# Implementation

A class is a good way to structure this. We initialize it with the array, it computes the prefix sum internally, and then it can answer multiple queries.

public class NumArray { private int[] prefixSum; public NumArray(int[] nums) { if (nums == null || nums.Length == 0) return; prefixSum = new int[nums.Length]; prefixSum[0] = nums[0]; for (int i = 1; i < nums.Length; i++) { prefixSum[i] = prefixSum[i-1] + nums[i]; } } public int SumRange(int left, int right) { if (prefixSum == null || left < 0 || right >= prefixSum.Length || left > right) { // Or throw an exception return 0; } if (left == 0) { return prefixSum[right]; } return prefixSum[right] - prefixSum[left - 1]; } }

4. Practice: Subarray Sum Problems

Prefix sums are crucial for solving problems like "Find a continuous subarray that sums to a target value `k`." (e.g., LeetCode 560. Subarray Sum Equals K).

The key insight is that the sum of a subarray from index `i` to `j` is `prefixSum[j] - prefixSum[i-1]`. So, we are looking for two indices `i` and `j` such that:

prefixSum[j] - prefixSum[i-1] = k

Rearranging this, we get:

prefixSum[i-1] = prefixSum[j] - k

This turns the problem into: "As we iterate through the array building the current prefix sum (`prefixSum[j]`), have we previously seen a prefix sum equal to `currentPrefixSum - k`?" A `Dictionary` or `HashSet` is perfect for storing the prefix sums we've seen so far, allowing for an O(1) lookup. This leads to an overall O(n) solution.

C# Implementation (Count Subarrays with Sum K)

public int SubarraySum(int[] nums, int k) { int count = 0; int currentSum = 0; // Dictionary to store prefix sums and their frequencies Dictionary prefixSumFreq = new Dictionary(); // Base case: a sum of 0 has been seen once (the empty prefix) prefixSumFreq[0] = 1; foreach (int num in nums) { currentSum += num; // Check if (currentSum - k) exists in the dictionary int complement = currentSum - k; if (prefixSumFreq.ContainsKey(complement)) { count += prefixSumFreq[complement]; } // Add the current prefix sum to the dictionary if (prefixSumFreq.ContainsKey(currentSum)) { prefixSumFreq[currentSum]++; } else { prefixSumFreq[currentSum] = 1; } } return count; }
